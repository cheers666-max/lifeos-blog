很多人讲 DPO 一上来就甩损失函数,看着像天书。这篇换个法子,**自顶向下、一条主线串到底**:先想清楚我们到底在解什么问题,再一步步把那条损失"长"出来。顺序参考了 Umar Jamil 的《DPO Explained》。每个公式都配大白话和数字,配手绘小黑图,零基础也能跟下来。

## 这趟要走的路(先记住这条主线)

后面每一节都是这条主线上的一步,读的时候时不时回来对一下位置,就不会迷路:

1. 想让模型**懂事**(对齐)→ 得先能**给回答打分**(奖励);
2. 可人给不出绝对分,只会说"**这个比那个好**"(偏好)→ 用 **Bradley-Terry** 把偏好翻译成分数;
3. 但只追高分,模型会**作弊**(reward hacking)→ 给目标加一条"**别跑太远**"的拴绳:目标 = 高分 − β·跑偏;
4. 解这个目标 → 得到"理想模型"该长什么样,可式子里卡着一个**算不出来的 Z**;
5. 神操作:两个回答**一相减,Z 自己没了** → 奖励变成"模型对这句话的态度变化" → 套进概率和惩罚,就是 **DPO 损失**。

一句话:**DPO 就是把"训练奖励模型 + 跑强化学习"这一大套,坍缩成一条能直接优化的损失。** 下面一步步看它怎么坍缩的。

## 0 · 先把符号钉死(随时回来翻)

| 符号 | 读法 | 它是什么 |
|---|---|---|
| `x` | — | 问题 / prompt |
| `y_w` | y-winner | 好回答(你圈「赢」的那个) |
| `y_l` | y-loser | 差回答(你圈「输」的那个) |
| `π_θ` | pi-theta | **正在训练的模型**(策略) |
| `π_ref` | pi-ref | **参考模型**:训练前冻结的原模型,当「拴绳」 |
| `r(x,y)` / `r*` | reward | 给「问题+回答」打的分(`*`=理想的那个) |
| `margin` | — | 好回答的奖励 − 差回答的奖励(天平指针) |
| `β` | beta | 拴绳硬度(大=保守,小=激进) |
| `σ(z)` | sigmoid | 把任意分数压成 0~1 的概率 |
| `Z(x)` | 配分函数 | 归一化项(算不出来,但会抵消) |

> 上标 `*` 一律读作「理想 / 真实的那个」(`r*`、`π*`)——是我们想逼近的目标,不是已知量。不用背,用到时再回来查。

## 1 · 大局:语言模型 + 对齐是什么(主线第 0 步)

先把最大的背景立住。**语言模型在干嘛?** 一句话:看着前面的文字,**预测下一个最可能的词(token)**,吐出来,再接着预测下一个,像超级自动补全,一个字一个字往外蹦。

**那为什么要"对齐"?** 因为"会补全下一个字"不等于"会好好回答人"。原始模型可能胡说、答非所问、学了网上的脏东西。**对齐(alignment)就是把模型调成人更想要的样子**——有用、诚实、无害。DPO 就是做对齐的一种办法。

它具体怎么做?用一句话加一张图说清:

**DPO 从头到尾就干一件事:拿人对一对回答的好恶,直接把模型往「好回答」那边掰。** 想象一台天平,好回答搁左盘、差回答搁右盘,指针偏多少叫 **margin**;DPO 一直在把指针往好回答那边推。这台天平是全文的核心画面,后面反复会用到它。

{{< figure src="01-balance-scale.png" alt="小黑把天平指针往好回答那边推" caption="小黑站在天平柱上把指针往「好回答」推——这就是 DPO 在做的唯一一件事" >}}

> **小结**:对齐 = 把模型调成人想要的样子;DPO 的做法 = 用偏好把天平指针往好回答推。**那"指针该往哪推、推多少",得有个标准——这个标准就是"奖励"。下一步先说清为什么需要它。**

## 2 · 为什么需要「奖励」,以及它带来的麻烦(主线第 1 步)

要把模型往"好回答"调,第一步得能**评判**回答好不好——也就是给回答**打个分**,这个分就叫**奖励(reward)**。有了分,才知道往哪个方向调。

怎么用这个分去调模型?**强化学习(RL)的思路**是:把模型看成一个**策略**,它生成回答,我们让它**多生成高分回答、少生成低分的**。听起来很顺。

**但这里有个大坑,叫 reward hacking(钻奖励的空子)。** 如果模型只盯着"把分刷高",它会去**作弊**:输出讨好评分、啰嗦、甚至胡编但"看着像高分"的垃圾。就像学生只为考分,会背套路、写正确的废话,而不是真懂。

所以光有"奖励"还不够,我们其实需要**两样东西**(记住这两样,后面所有数学都在实现它们):

1. **一个靠谱的打分器**——而且最好从"人容易给的判断"里学(下一节就讲为什么是偏好);
2. **一根拴绳**——别让模型为了刷分,变得离原来的样子太远(这就是后面会出现的 **β / 约束** 的由来)。

> **小结**:我们要"打分器 + 拴绳"。**这一节先欠着两笔账:打分器怎么造?拴绳怎么加?第 4 节还打分器,第 5 节还拴绳。** 但在造打分器之前,得先备三个小数学工具,否则公式看不懂——这就是下一节。

## 3 · 备三块数学积木:log / −log / sigmoid(过路工具箱)

这一节是个"工具箱中转站":后面的 Bradley-Terry 和损失函数都要用到 log、−log、sigmoid 这三样。先花两分钟搞懂它们各自在干嘛,用到时再回来查。先记一句免得踩坑:margin=0 这点藏着**两个不同的数**——`σ(0)=0.5` 是「概率」,`−log σ(0)≈0.69` 才是「损失」,而 `−log(0)` 是**正无穷**(不是 0.5!)。

{{< figure src="fig-log-sigmoid.png" alt="ln(x), -ln(p), sigmoid 三张曲线" caption="三块积木:① log 长什么样 ② −log(p) 当损失 ③ sigmoid 把分数压成概率" >}}

### 积木① — log:就是问「放大还是缩小了」

`log` 问的事特别朴素:**这个数是 1 的多少倍?用「放大了还是缩小了」来记。** 训练**前**某句话概率 0.2,**后**涨到 0.4,放大 `0.4÷0.2=2` 倍,`log(2)≈0.69` 正数——更想说了;掉到 0.1 是缩小一半,`log(0.5)≈−0.69` 负数——更不想说了;没变就是 `log(1)=0`。

> 一句话记牢:**log 大于 0 是放大、等于 0 是没变、小于 0 是缩小。** 这一条就够读懂全文所有的 log。

{{< figure src="02-log-ruler.png" alt="小黑在以1为中点的标尺上推概率球" caption="以「1」为中点的标尺:往右推=放大→正,往左=缩小→负" >}}

### 积木② — −log(p):念成「模型有多吃惊」

`p` 是模型给**正确答案**分配的概率,越接近 1 越好。怎么把「p 越大越好」翻成"越小越好的损失"?前面加个 `−log`,念成「这次有多吃惊」:

```text
p = 0.9  (笃定选对)      −log ≈ 0.11   几乎不罚
p = 0.5  (半信半疑)      −log ≈ 0.69   罚一点
p = 0.1  (笃定却选错)    −log ≈ 2.30   狠狠罚
p → 0    (打死认定错)    −log → +∞     罚到天上去
```

> 一句话:**−log(p) 就是"答错的罚单",对了不罚、越自信地错罚得越狠。** 这其实就是你天天用的交叉熵。

{{< figure src="03-penalty-hammer.png" alt="小黑举的惩罚锤随错得多自信而变大" caption="−log(p) 就是惩罚锤:答对不罚,自信答错锤头越来越大" >}}

### 积木③ — sigmoid:一台「压缩机」

模型内部算出来的常是个没边界的分数(可能 −5 也可能 +8)。`σ` 把它**压回 0~1 当概率**:`σ(0)=0.5`、`σ(2)≈0.88`、`σ(−1)≈0.27`。

> 一句话:**sigmoid 把"分数"翻译成"概率"。** 在 DPO 里,喂给它的那个分数,就是天平指针 margin。

三块工具备齐了,回到主线:造打分器。

## 4 · 造打分器:从偏好到分数 Bradley-Terry(主线第 2 步,还第一笔账)

第 2 节欠的第一笔账——打分器怎么造?难点是:**怎么得到分数?**

### 为什么不直接给回答打分?

- **绝对打分,难且不靠谱。** 让人给单个回答打「0~100 分」,因人而异、因时而异,「70 和 72 差在哪」说不清,既不稳定也不可比。
- **相对比较,简单又一致。** 让人选「两个里哪个更好」,认知负担低、结论稳。人天生擅长比大小,不擅长定绝对值。

所以真实流程是:**人只做容易的相对判断(A 比 B 好)→ 攒一堆这种对子 → 用 Bradley-Terry 把相对偏好"反推"成绝对分数。** Bradley-Terry(给棋手排名的 Elo 是它近亲)就是那台「把好收集的相对偏好,翻译成难直接得到的绝对分数」的机器。

### Bradley-Terry:把「谁赢」变成概率

它的想法:**给每个回答一个隐藏的「实力分」 `r*`,好回答赢过差回答的概率,只由两者实力分的差决定。**

$$
P(y_w \succ y_l) = \frac{e^{r^*(x,\,y_w)}}{e^{r^*(x,\,y_w)} + e^{r^*(x,\,y_l)}} = \sigma\big(r^*(x,y_w) - r^*(x,y_l)\big)
$$

*把两个 e^实力分 做 softmax,等价于把「实力分之差」喂给 sigmoid。*

公式有点唬人,先学会**读 `r*(x, y_w)` 这种写法**(很多人就卡这):它就是中学的 `f(x)`——括号前是函数名,括号里是塞进去的输入。

```text
   r*  (  x ,  y_w  )
   ↑       ↑     ↑
 函数名   塞进去的两样东西
(打分机)  ① 问题 x  ② 回答 y_w
```

`r*` 是一台**打分机**,把「问题+回答」塞进去吐一个分数。为什么连问题也要塞?因为同一个回答换个问题,好坏就变了。**用大白话读整个公式**:每个回答先按它的分算一个"权重"(分越高权重越大),好回答的权重 ÷ 两个权重之和,就是"好回答会赢"的概率。算一下上海那行(好回答 3 分、差回答 0 分):

```text
好回答权重 = e³ ≈ 20.1
差回答权重 = e⁰ = 1
P(好赢)   = 20.1 / (20.1 + 1) ≈ 0.95   → 好回答 95% 会赢
```

它本质就是个**两选一**。把上下同时除一下,这一坨 `e` 立刻塌成一个 sigmoid(积木③):

```text
P(好赢) = σ( 好回答分 − 差回答分 )
```

**这就是关键的衔接点:** 好坏两个回答的"分差",喂进 sigmoid,就是"好回答会赢的概率"。记住这句,等下损失函数里它会原样再出现一次。

### 再深一层:分差就是「赔率」(可选,加深直觉)

把上面那条反过来看赔率,直觉更牢:

$$\frac{P(\text{赢})}{P(\text{输})} = e^{\,\text{分差}}$$

也就是**分差 = 赔率的对数**;每拉开 1 分,赢的赔率就 ×e(≈2.7 倍):分差 1→约 2.7:1(73%)、分差 2→约 7.4:1(88%)、分差 3→约 20:1(95%)。还有更妙的一层:你给的全是零散的「A>B」,**BT 把它们拼成一条统一的"分数尺"**,每个回答在尺上有唯一位置(像从一堆"谁比谁高"反推每个人的身高)。

{{< figure src="09-bt-ruler.png" alt="小黑把一堆 A>B 卡片拼成一条分数尺" caption="BT 的活儿:把散乱的「A>B」拼成一条统一分数尺,r* 就是每个回答在尺上的位置" >}}

那这套分数 `r*` 怎么定下来?用**最大似然估计**:调分数,让"你标的偏好全被排对"的概率最大。这一步训出来的东西,就叫**奖励模型**——RLHF 的常规做法正是先训它,再拿它去跑强化学习。

### 算一遍:从一堆战绩,反推出实力排名

光说没感觉,代一组真数字。设三个选手两两对战:**A 对 B 8 胜 4 负;A 对 C 3 胜 5 负**。给每人一个实力分 $\alpha>0$,胜率 = 自己分 ÷ 两人分之和(这就是 BT 的 $\alpha$ 版,和前面 $\sigma$ 版是一回事:令 $\alpha=e^{r}$ 就有 $\tfrac{\alpha_i}{\alpha_i+\alpha_j}=\sigma(r_i-r_j)$)。BT 要找的,是让"这些战绩最可能发生"的那组 $\alpha$,也就是**最大化对数似然 $\ln L$**。

因为 B、C 都只和 A 打,可以直接解出来(固定 $\alpha_A=1$ 当基准):

$$
\begin{aligned}
\text{A 对 B 胜率} &= \tfrac{8}{12}=0.667 \;\Rightarrow\; \alpha_B=\alpha_A\cdot\tfrac{4}{8}=0.5 \\
\text{A 对 C 胜率} &= \tfrac{3}{8}=0.375 \;\Rightarrow\; \alpha_C=\alpha_A\cdot\tfrac{5}{3}=1.667
\end{aligned}
$$

代进对数似然,一行行化简:

$$
\begin{aligned}
\ln L &= 8\ln\tfrac{\alpha_A}{\alpha_A+\alpha_B} + 4\ln\tfrac{\alpha_B}{\alpha_A+\alpha_B} + 3\ln\tfrac{\alpha_A}{\alpha_A+\alpha_C} + 5\ln\tfrac{\alpha_C}{\alpha_A+\alpha_C} \\
      &= 8\ln 0.667 + 4\ln 0.333 + 3\ln 0.375 + 5\ln 0.625 \\
      &= -3.24 - 4.40 - 2.94 - 2.35 \\
      &= -12.93
\end{aligned}
$$

逐项代入看得更清楚:

| 对局 | 公式项 | 代入 | 结果 |
|---|---|---|---|
| A 赢 B ×8 | 8·ln(α_A/(α_A+α_B)) | 8·ln 0.667 | −3.24 |
| B 赢 A ×4 | 4·ln(α_B/(α_A+α_B)) | 4·ln 0.333 | −4.40 |
| A 赢 C ×3 | 3·ln(α_A/(α_A+α_C)) | 3·ln 0.375 | −2.94 |
| C 赢 A ×5 | 5·ln(α_C/(α_A+α_C)) | 5·ln 0.625 | −2.35 |
| **合计** | | | **ln L = −12.93** |

对照"瞎猜三人一样强"(每人 α=1、每场胜率都 0.5):$\ln L = 20\times\ln 0.5 = -13.86$,明显更差。所以上面那组 α 更贴合战绩——这就是最大似然在挑的东西。

**推出的实力排名:$\alpha_C(1.667) > \alpha_A(1) > \alpha_B(0.5)$。** 注意:A 把 B 打得挺惨,可 C 把 A 压着打,所以**最强是 C**。从一堆两两胜负,反推出一条全局实力尺——这正是 Bradley-Terry "把偏好变成分数"的本事。

> **小结(打分器到手)**:偏好 →(Bradley-Terry)→ 分数 `r*`;而"好坏分差 → σ → 赢的概率"这条会在损失里重现。**第一笔账还清了。但 DPO 的野心是连这个奖励模型都不单独训——别急,先还第二笔账:那根拴绳。**

## 5 · 加拴绳:RLHF 的真正目标 = 最大奖励 − β·拴绳(主线第 3 步,还第二笔账)

回到第 2 节欠的第二笔账。光追高分会 reward hacking,所以要给目标**加一条"别跑太远"的拴绳**。把这两句话写成一个目标函数,就是 RLHF / DPO 共同的出发点:

```text
让模型      多拿奖励          但别离原模型太远
  ↓            ↓                    ↓
 max    E[ 奖励 r(y) ]    −    β · (跑偏程度)
```

**用大白话**:既要回答得分高,又别变得太离谱;中间用 `β` 调这两者的平衡。那个"跑偏程度"有个正式名字叫 **KL 散度**,就是"现在的模型 π_θ 离原模型 π_ref 有多远"的一把尺,两者一样时为 0。而 **β 就是这根拴绳的硬度**——正是第 2 节埋的伏笔。

{{< figure src="08-beta-knob.png" alt="小黑拧 β 旋钮,橡皮筋时紧时松" caption="β 大→绷紧,贴着原模型;β 小→松,撒丫子跑向高奖励(太小就 reward hacking 跑飞)" >}}

| β | 会发生什么 |
|---|---|
| **大** | 拴绳太硬,几乎不动、学不进偏好 → **太大 = 学不动** |
| **小** | 为冲高奖励使劲扭曲 → **太小 = 跑飞**(复读、退化、reward hacking) |
| **常用** | 多数实现默认 `β=0.1`,区间约 `0.1~0.5` |

> **小结**:目标 = **高分 − β·跑偏**;β 大保守、小激进,reward hacking 就发生在 β 太小时。**两笔账都还清了(打分器 + 拴绳)。接下来:把这个目标解出来,看"理想模型"长什么样。**

## 6 · 解这个目标:理想模型长啥样,以及那个算不出的 Z(主线第 4 步)

上面那个"高分 − β·跑偏"是带约束的优化,数学上能**直接解出来**,答案出奇地简洁——

**理想模型 = 拿原模型 π_ref,把"高奖励的回答"按比例加厚,再重新凑成总和为 1。**

$$
\pi^*(y\mid x) = \frac{1}{Z(x)}\,\pi_{ref}(y\mid x)\,e^{\,r(x,y)/\beta}
$$

*最优策略 = π_ref 按奖励重新分堆(乘 e^(r/β)),再除以 Z 归一化。*

公式里 `e^(r/β)` 就是"按奖励加厚"的倍数(奖励越高、加得越厚),`Z` 只是把加厚后的概率重新凑成 1 的归一化分母。小黑把这事画成了"按奖励重新分豆子":

{{< figure src="06-bean-tilt.png" alt="小黑把原模型一袋豆子按奖励重新分堆" caption="小黑把 π_ref 这袋豆子按奖励重新分堆:奖励越高堆越大,β 越小分得越极端" >}}

拿数字感受一下 β 怎么影响"分得多狠"(原模型 `[0.5,0.3,0.2]`、奖励 `[0,1,2]`):

```text
β = 5   →  [0.43, 0.31, 0.26]   几乎没动,还是原模型(保守)
β = 1   →  [0.18, 0.29, 0.53]   开始偏向高奖励
β = 0.2 →  [0.00, 0.01, 0.97]   孤注一掷压最好那个(太小=跑飞)
```

这正好印证第 5 节:**β 越小越激进**。现在把这条"理想模型"反过来,用它表示**奖励**(两边取 log、移项即可):

$$
r(x,y) = \beta\log\frac{\pi^*(y\mid x)}{\pi_{ref}(y\mid x)} + \beta\log Z(x)
$$

*奖励 = β·log(π\*/π_ref) + β·logZ。前半截是隐含奖励,后半截 βlogZ 是个零头。*

**用大白话**:一个回答的奖励 = `β × log(理想模型概率 ÷ 原模型概率)`,再加一个零头 `β·logZ`。前半截特别好——它就是"模型现在比原来多想说这句多少"。坏就坏在那个 `Z`:

> ⚠️ **Z 要对「模型能生成的所有句子」求和**——句子有无穷多种,**这玩意儿根本算不出来**。早期方法就卡死在这里。

> **小结**:理想模型 = 原模型按奖励加厚;反解出"奖励 = β·log(概率比) + 一个算不出的 Z"。**眼看要死在 Z 上了——下一节就是全文最妙的一步:让 Z 自己消失。**

## 7 · 临门一脚:Z 自己消失 → DPO 损失(主线第 5 步,收口)

DPO 的神来之笔特别简单:**不去算 Z,只比较同一个问题下的两个回答。** 因为那个 Z 只跟"问题"有关、跟"具体回答"无关,所以两个回答的奖励**一相减,Z 就一模一样地抵消掉了**:

```text
好回答奖励 − 差回答奖励
 = [ β·log(好回答概率比) + β·logZ ]
 − [ β·log(差回答概率比) + β·logZ ]
                          └──┬──┘
              同一个问题,这两块完全一样 → 一减为零!
 = β·log(好回答概率比) − β·log(差回答概率比)
 = margin   ← 又见天平指针!
```

{{< figure src="07-z-cancel.png" alt="小黑把两边相同的 Z 划掉" caption="两个回答各顶一块一样的 Z,一相减,小黑一笔划掉——算不出也没关系" >}}

Z 一抵消,**奖励就不用单独训练了**,它直接等于 `β·log(π_θ/π_ref)`——大白话就是"**模型现在比从前多想说这句话多少**",一支量态度的温度计:

$$
r(x,y) = \beta\log\frac{\pi_\theta(y\mid x)}{\pi_{ref}(y\mid x)}
$$

*隐含奖励 = β·log(训练后概率 ÷ 训练前概率)。*

{{< figure src="04-reward-thermometer.png" alt="小黑把温度计插进对话气泡读数" caption="奖励 = 态度温度计:更想说→升,更不想说→降" >}}

现在把所有零件拼起来,主线收口。还记得第 4 节那句"**分差 → σ → 赢的概率**"、和第 3 节那句"**−log 是答错的罚单**"吗?两句一合,就是 DPO 损失:

```text
① 两个回答的"分差"      = margin = β·log(好的概率比) − β·log(差的概率比)
② 喂进 sigmoid           → 得到"好回答会赢"的概率        (第 4 节)
③ 这概率越大越好 → 取 −log 当损失,越小越好               (第 3 节)
                ─────────────────────────────
   DPO 损失 = −log σ( margin )
```

$$
\mathcal{L}_{DPO} = -\log\sigma\!\left(\beta\log\frac{\pi_\theta(y_w\mid x)}{\pi_{ref}(y_w\mid x)} - \beta\log\frac{\pi_\theta(y_l\mid x)}{\pi_{ref}(y_l\mid x)}\right)
$$

*括号里 = margin(天平指针);外面 −logσ 负责「指针不够偏就狠罚」。*

**真拿数字走一遍**(取 β=1)。一对"模型已经学得不错的":好回答训练后概率 0.4(原来 0.2)、差回答 0.1(原来 0.2):

$$
\begin{aligned}
h_w &= \log\tfrac{0.4}{0.2} = \log 2 = 0.69, \qquad h_l = \log\tfrac{0.1}{0.2} = \log 0.5 = -0.69 \\
\text{margin} &= h_w - h_l = 0.69-(-0.69) = 1.38 \\
\sigma(1.38) &\approx 0.80 \quad(\text{模型觉得好回答 80\% 会赢}) \\
\text{损失} &= -\log\sigma(1.38) = -\log 0.80 \approx 0.22 \quad(\text{不大,基本学会了})
\end{aligned}
$$

换一对"还没学会的"($h_w=0.10,\ h_l=0.05$):

$$
\text{margin}=0.05 \;\Rightarrow\; \sigma(0.05)\approx 0.51 \;\Rightarrow\; \text{损失}=-\log 0.51 \approx 0.67
$$

明显更大。**同一条公式,会的罚得轻、不会的罚得重**,训练的劲儿自动跑去收拾还没学会的对子。

> **小结(主线收口)**:Z 在成对相减里自己没了 → 奖励变成 `β·log(π_θ/π_ref)` → 套进 σ 和 −log 就是 DPO 损失。**一个本来要"训奖励模型 + 跑 RL + 估 Z"的大流程,坍缩成一条监督式损失——这就是名字里 Direct 的来历。** 顺带两个彩蛋:**为什么必须成对?** Z 的抵消靠的就是同一问题下两回答相减;**为什么 SimPO 敢扔掉 π_ref?** 它换了套奖励定义,不走 Z-抵消这条路了。

## 8 · 两个反直觉的坑(用前面学的回头看)

**坑 A — 越不会的,推得越狠。** 训练时每对子的"推力"(梯度大小)= `σ(−margin)`:

```text
学得不错那对:  margin = 1.38   推力 ≈ 0.20   轻轻推
还没学会那对:  margin = 0.05   推力 ≈ 0.49   使劲推
要是学反了:    margin = −2     推力 ≈ 0.88   往死里推
```

{{< figure src="fig-gradient-vs-margin.png" alt="损失与推力随 margin 的变化" caption="推力 σ(−margin):模型越错推得越狠,学会了就放手" >}}

这里有个**反复强调过、但极易混的点**:**标签**是你定死的(谁赢谁输);**margin** 是模型实时算的(它眼下觉得好回答比差回答好多少)。所以 margin≈0 **不是**"标签模糊",而是"模型还没调到同意你的标注"。DPO **无条件相信你的标签**——你标错了它照样理直气壮地学错,这正是 cDPO、IPO 后来要打折扣的地方。

**坑 B — 好回答的概率,其实可能在往下掉。** DPO 靠"拉开差距"满足损失,**不保证好回答的绝对概率真涨**。常见的是好坏概率一起往下掉、只是差的掉更快——差距照样拉开了。这就是有人给 DPO 加 SFT 正则(如 DPOP)摁住好回答的原因。

{{< figure src="05-both-sink.png" alt="好回答和差回答一起掉进洞,差的掉更快" caption="坑 B:两块一起往下掉,只是差的掉得更快——好回答的绝对概率也跌了" >}}

## 9 · 落地:公式里的 log 概率到底怎么算

最后落到能上手的一步。公式里反复出现 `log π(y|x)`——它是「**整句回答**」的对数概率。一句话有很多字,**整句概率 = 每个字概率连乘**,取 log 后就是**连加**(这就是积木①里"log 把连乘变连加"的用处):

```text
log π(整句回答 | 问题) = 把答案里每个字的 log 概率加起来
```

具体做法:因为答案是数据集里**给定的**,每个字是什么我们都知道;把"问题 + 答案"喂进模型,逐个取出"模型在那一步给这个真实字的对数概率",求和,就是这条回答的 `log π(y|x)`。同一句话在 `π_θ`(训练中)和 `π_ref`(冻结)上各算一遍、相减,就是隐含奖励里那个 `log(π_θ/π_ref)`;好回答、差回答各来一遍,一减就是 margin。整个 DPO 训练,就是在大量这样的成对样本上,把 `−log σ(margin)` 降下去。

## 10 · 白月光调查档案:把 DPO 当成你要查的「人」

读到这里,与其把 DPO 当一堆要背的公式,不如把它当成**一个你要扒的人**——这就是「白月光学习法」的思路:**别背知识,去调查它。** 你能花三小时扒前任的关系网,就能用同样的劲头扒清一个概念。给 DPO 建个档案,它的来龙去脉一下就立体了:

- **它是谁**:一种对齐方法,用成对偏好**直接**训练模型,跳过"单独训一个奖励模型"。
- **它从哪来**:老 RLHF(训奖励模型 + 跑 PPO 强化学习)又重又易崩;2023 年 DPO 把这一整套**数学上坍缩成一条损失**。
- **如果它消失会怎样**:你就得退回"训奖励模型 + 跑 RL + 估那个算不出的 Z"的老路——工程复杂、还容易 reward hacking。它的价值,正是消去了这些。
- **它和谁有关(关系网)**:上游缝合了四样——Bradley-Terry(偏好→分数)、RLHF 目标(高分 − β·拴绳)、KL 约束、sigmoid/交叉熵。本文前 7 节其实就是在一根根接上这张网。
- **谁依赖它(它的"粉丝")**:IPO(给指针装限位器)、KTO(改单点、不要成对)、SimPO(扔掉 π_ref)、cDPO(标签平滑)、online DPO / self-play——整个家族都站在它肩上。
- **下一条线索**:顺着"它哪里不够好"接着查——嫌它**对噪声脆弱** → 查 IPO / cDPO;嫌**成对数据贵** → 查 KTO;嫌 **π_ref 占显存** → 查 SimPO。

{{< figure src="10-investigation-board.png" alt="小黑当侦探,在板子上用红线把 DPO 和相关概念连起来" caption="把概念当成要查的人:DPO 在中心,红线连到 Bradley-Terry / RLHF / KL,再牵出下一条线索 IPO / KTO / SimPO" >}}

> **这套调查法你可以套到任何概念上**,问七个问题就行:它是谁?从哪来?为什么存在?和谁有关?谁依赖它?如果它消失?下一条线索是什么?——调查会激活好奇心,背诵只会激活痛苦。

## 收尾:把主线再串一遍

> **想让模型懂事(对齐)→ 得给回答打分(奖励);可人只会说"这个比那个好"(偏好)→ Bradley-Terry 把偏好翻成分数;只追高分会作弊(reward hacking)→ 给目标加拴绳:高分 − β·跑偏;解这个目标得到"理想模型",里面卡着一个算不出的 Z;但两个回答一相减,Z 自己没了 → 奖励变成 β·log(π_θ/π_ref) → 套进 σ 和 −log,就是 DPO 损失。Direct 的意思,就是把奖励模型直接吸收进了损失。**
