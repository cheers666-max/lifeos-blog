很多人讲 DPO(Direct Preference Optimization)一上来就甩损失函数,看着像天书。这篇我换个法子:先把画面立住,再备齐三块数学小积木,然后把损失一块一块拼出来——每个公式都带数字给你算一遍。读完你应该能自己把 `−logσ(margin)` 从头讲给别人听,还能说清那个"DPO 不用训奖励模型"的魔术到底是怎么变的。

> 这篇属于「偏好优化家族」系列的第一篇。后面会接着写 IPO、KTO、SimPO,看每个变体到底动了哪个零件。

## 1 · 先把画面立住:一台天平

你手里有一沓卡片,每张卡上是同一个问题的两个回答——一个你圈了「这个好」(叫它 `y_w`,winner),一个「这个差」(`y_l`,loser)。把它想象成一台天平:好回答搁左盘,差回答搁右盘,指针偏多少,就叫 **margin**。

**DPO 这一整套东西,从头到尾就干一件事:把指针往「好回答」那边掰过去。** 后面那些公式看着唬人,其实全都是在把这句话翻译成机器能去优化的数学而已。

## 2 · 备料:三块数学小积木

公式之所以像天书,通常不是因为它难,而是里头的 log、σ 你没真正搞懂它们各自在干嘛。先记一句话免得踩坑:在 margin=0 这点上藏着**两个不一样的数**,很多人会混——`σ(0)=0.5` 是「概率」,`−log σ(0)=−log(0.5)≈0.69` 才是「损失」,而 `−log(0)` 是**正无穷**(不是 0.5!)。那个 0.5 是 sigmoid 算的,跟 log 没关系。

{{< figure src="fig-log-sigmoid.png" alt="ln(x), -ln(p), sigmoid 三张曲线" caption="三块积木:① log 长什么样 ② −log(p) 当损失 ③ sigmoid 把分数压成概率" >}}

### 积木① — log,其实就是问「放大还是缩小了」

`log` 这名字听着吓人,它问的事特别朴素:**这个数是 1 的多少倍?用「放大了还是缩小了」来记。**

举个例子。训练**前**模型说某句话的概率是 0.2(参考模型 π_ref),训练**后**涨到 0.4(π_θ),那它放大了 `0.4 ÷ 0.2 = 2` 倍,`log(2) ≈ 0.69`,正数——**正,代表比以前更想说了**。反过来掉到 0.1,是 `÷2`,`log(0.5) ≈ −0.69`,负数——**更不想说了**。没变就是 `log(1)=0`。

> 记住这一句就够读懂 DPO 里所有的 log:**>0 是放大,=0 没变,<0 是缩小,趋近 0 时一路掉到负无穷。**

{{< figure src="02-log-ruler.png" alt="小黑在以1为中点的标尺上推概率球" caption="以「1」为中点的标尺:往右推=放大→正,往左=缩小→负" >}}

那为啥模型里到处取 log?因为它把**连乘变连加**(`log(a×b)=log a + log b`)。一句话几十个字,整句概率是每个字连乘,小到电脑都快表示不了;取个 log 变连加,不下溢、梯度还好算。这就是为什么写「对数概率 log π」而不是 π。

### 积木② — −log(p),把它念成「模型有多吃惊」

`p` 是模型给**正确选项**分配的概率,我们希望它越接近 1 越好。可优化器只会往小了降,怎么把「p 越大越好」翻成损失?加个 `−log` 就行,直接念成「模型这次有多吃惊」:

```text
p = 0.9  (笃定选对)      −ln(0.9) ≈ 0.11   几乎不罚
p = 0.5  (半信半疑)      −ln(0.5) ≈ 0.69   罚一点
p = 0.1  (笃定却选错)    −ln(0.1) ≈ 2.30   狠狠罚
p → 0    (打死认定错的)  −ln(0)   → +∞     罚到天上去
```

负号是因为 p∈(0,1) 时 log(p) 永远为负,加负号翻成正损失。**这就是你天天在用的交叉熵**——对了不罚,越自信地答错罚得越狠。

{{< figure src="03-penalty-hammer.png" alt="小黑举的惩罚锤随错得多自信而变大" caption="−log(p) 就是惩罚锤:答对不罚,自信答错锤头越来越大" >}}

### 积木③ — sigmoid,一台压缩机

模型内部算出来的往往是个没边界的分数(可能 −5 也可能 +8)。`σ(z)=1/(1+e^(−z))` 就是台压缩机,把任意分数压回 0~1 当概率:`σ(0)=0.5`、`σ(2)≈0.88`、`σ(−1)≈0.27`。在 DPO 里,喂给它的分数 z 就是 margin。

## 3 · 奖励是哪来的?

DPO 老说自己「不用训奖励模型」,可损失里又冒出个「奖励」。老 RLHF 是另训一个模型打分;DPO 发现:在「别跑太偏」(KL 约束)这个条件下,最优策略和奖励之间有个能写死的关系,反解出来,奖励就等于:

{{< figure src="fig-implicit-reward.png" alt="r = beta log (pi_theta / pi_ref)" caption="r = β × log(训练后概率 ÷ 训练前概率)。奖励藏在两个模型的概率比里" >}}

拿积木①翻译:括号里是「训练后÷训练前」,即模型对这句话的态度变了多少倍;外面 log 变成带符号的数;β 是个旋钮,管这事算多大分量。**一句话的奖励 = 模型现在比从前多想说它多少。奖励不是外请的裁判,是模型自己态度的温度计。**

{{< figure src="04-reward-thermometer.png" alt="小黑把温度计插进对话气泡读数" caption="奖励 = 态度温度计:更想说→升,更不想说→降" >}}

## 4 · 拼起来,然后真给你算一对

拼之前先补一块桥:我们手里只有「A 比 B 好」这种**成对偏好**,可训练要的是**数字分数**。把偏好变成分数的经典办法,叫 **Bradley-Terry 模型**(给棋手/球队排名的 Elo 评分就是它的近亲)。

### 先认识 Bradley-Terry:把「谁赢」变成概率

它的想法特别朴素:**给每个回答安一个隐藏的「实力分」 `r*`,然后"好回答赢过差回答的概率"只由两者实力分的差决定。**

{{< figure src="fig-bradley-terry.png" alt="Bradley-Terry 公式及其等于 sigmoid 的形式" caption="把两个 e^实力分 做 softmax,等价于把『实力分之差』喂给 sigmoid" >}}

别被 `e` 吓到,它就是个**两选一的 softmax**:分母是两个回答各自 `e^实力分` 的和(总盘子),分子是好回答那份,相除就是它赢的比例。套 `e` 有两个好处——实力分可正可负、套完恒为正能当概率;而且让「**只有差值说了算**」。上下同除 `e^{r_w}` 你就看见它其实就是 sigmoid:

```text
P = e^{r_w} / (e^{r_w} + e^{r_l})
  上下同除 e^{r_w}
  = 1 / (1 + e^{−(r_w − r_l)})
  = σ( r_w − r_l )          ← 「好回答实力分 − 差回答实力分」喂给 sigmoid
```

这正是下面要用的 `σ(margin)`。算几个数感受一下「只认差」:

```text
r_w=2, r_l=1 → P = e²/(e²+e¹) = 0.73   (= σ(1),挺有把握赢)
r_w=1, r_l=1 → P = 0.50                 (平手)
r_w=1, r_l=3 → P = e/(e+e³)  = 0.12     (= σ(−2),反而多半输)
把第一组都 +100 → P 还是 0.73           (只有差 r_w−r_l 起作用,整体平移不影响)
```

> 最后这条「整体平移不影响」很重要——它正是后面第 7 节里 **βlogZ 在成对相减中抵消** 的同一个道理:BT 只认差,常数自己消掉。
>
> 那 `r*` 怎么定?用**最大似然估计(MLE)**:调整这套分数,让"你标的所有偏好都被排对"的总概率最大。把每对的 `σ(r_w−r_l)` 连乘要最大,等价于最小化 `Σ −logσ(r_w−r_l)`——**这条就是 RLHF 里训练奖励模型的损失**。而 DPO 的偷懒之处,就是把 `r*` 直接换成 `β·log(π_θ/π_ref)`(下一步的隐含奖励),省掉单独训奖励模型这一整步。

现在积木齐了,拼:

```text
① 我们要的:    让「赢家干掉输家」的概率尽量大
② Bradley-Terry: 赢家赢的概率 = σ(赢家分 − 输家分) = σ(margin)   ← 积木③
③ 每个人的分:   就是隐含奖励,两人一减 = margin                 ← 积木①
④ 变成损失:     想让概率大,就对它取 −log                       ← 积木②
                ─────────────────────────────────
   L = −log σ(margin) = −log σ(β(h_w − h_l))
```

{{< figure src="fig-dpo-loss.png" alt="DPO 损失函数" caption="括号里=margin=好回答奖励−差回答奖励;外面 −logσ 负责「差得不够大就狠罚」" >}}

真拿数字走一遍(先把 β 设成 1)。**第一对(已经学得不错的):** 赢家训练后 0.4 训练前 0.2,输家训练后 0.1 训练前 0.2:

```text
h_w = log(0.4/0.2) = log(2)   =  0.69
h_l = log(0.1/0.2) = log(0.5) = −0.69
margin = 0.69 − (−0.69) = 1.38
σ(1.38) ≈ 0.80   → 模型觉得赢家有 80% 会赢,挺笃定
损失 = −ln(0.80) ≈ 0.22   → 不大,基本学会了
```

**第二对(还没学会的):** `h_w=0.10, h_l=0.05`:

```text
margin = 0.05
σ(0.05) ≈ 0.51   → 几乎五五开
损失 = −ln(0.51) ≈ 0.67   → 明显更大
```

同一条公式,「已经会的」罚得轻、「还不会的」罚得重——训练的劲儿自动跑去收拾还没学会的对子,不用手动安排。

## 5 · β:橡皮筋的硬度

分母里的 π_ref 是根「橡皮筋」,拴着 π_θ 别跑太远。**β 就是这根橡皮筋的硬度。** 还有个等价说法:DPO 想要的最优结果,其实就是「拿原模型,按奖励重新分堆」,而 β 控制分得多狠。

{{< figure src="08-beta-knob.png" alt="小黑拧 β 旋钮,橡皮筋时紧时松" caption="β 大→绷紧,贴着原模型;β 小→松,撒丫子跑向高奖励(太小会跑飞)" >}}

用一个例子扫一遍 β(原模型 `[0.5,0.3,0.2]`,奖励 `[0,1,2]`,最优解是 π_ref 乘 `e^(r/β)` 再归一):

```text
β = 5   →  π* ≈ [0.43, 0.31, 0.26]   几乎没动,还是原模型
β = 1   →  π* ≈ [0.18, 0.29, 0.53]   开始偏向高奖励
β = 0.5 →  π* ≈ [0.04, 0.16, 0.80]   重重压在最好那个上
β = 0.2 →  π* ≈ [0.00, 0.01, 0.97]   孤注一掷
```

{{< figure src="fig-beta-effect.png" alt="不同 beta 下的损失曲线和最优分布" caption="左:β 越大损失曲线越陡(小动作就满意→贴 ref);右:β 越小概率越堆向高奖励" >}}

| β | 会发生什么 |
|---|---|
| **大**(→5) | 橡皮筋硬,几乎不动、学不进偏好。**太大 = 学不动** |
| **小**(→0.2) | 概率全压高奖励,激进。**太小 = 跑飞**(复读、退化、reward hacking) |
| **常用** | 多数实现默认 `β=0.1`,区间约 `0.1~0.5` |

> 最容易记反的一点:β 在 `e^(r/β)` 的**分母**,所以「**越小越猛**」。一句话:β 大=贴原模型(保守),β 小=冲高奖励(激进)。

## 6 · 两个反直觉的坑

**坑 A — 越不会的、推得越狠。** 推力(梯度大小)= `σ(−margin)`:

```text
学得不错那对:  margin = 1.38   推力 = σ(−1.38) ≈ 0.20   轻轻推
还没学会那对:  margin = 0.05   推力 = σ(−0.05) ≈ 0.49   使劲推
要是学反了:    margin = −2     推力 = σ(2)     ≈ 0.88   往死里推
```

{{< figure src="fig-gradient-vs-margin.png" alt="损失与推力随 margin 的变化" caption="推力 σ(−margin):模型越错推得越狠,学会了就放手" >}}

这里有个特别容易混的点:**标签**是你定的、钉死不动的(谁赢谁输);**margin** 是模型实时算的(它眼下觉得赢家比输家好多少)。所以 margin≈0 **不是**「标签模糊」,而是「模型还没调整到同意你的标注」。DPO **无条件相信你的标签、没有『真实类别』的概念**——你标错了它照样理直气壮地学错。这正是 cDPO、IPO 要给「无条件信任」打折的原因。

**坑 B — 好回答的概率,其实可能在往下掉。** DPO 满足损失靠「拉开差距」,不保证 `π_θ(y_w)` 真的涨。常见的是好坏概率一起掉、只是差的掉得更快——差距照样拉开。这就是为什么有人给 DPO 加 SFT 正则(如 DPOP)摁住好回答。

{{< figure src="05-both-sink.png" alt="好回答和差回答一起掉进洞,差的掉更快" caption="坑 B:两块一起往下掉,只是差的掉得更快——差距拉开了,好回答的绝对概率也跌了" >}}

## 7 · 魔术揭秘:那个算不出来的 Z,怎么消失的

前面说奖励是「反解」出来的。现在把窗户纸捅破。RLHF 的目标是:

```text
max   E[ 奖励 r(y) ]   −   β · KL(π ‖ π_ref)
 π      ↑ 想要高奖励         ↑ 但别离原模型太远
```

这个问题有闭式解——**拿 π_ref,给每个回答乘奖励权重 `e^(r/β)`,再除以 Z 归一化:**

{{< figure src="fig-kl-solution.png" alt="pi star = 1/Z pi_ref exp(r/beta)" caption="最优策略 = π_ref 按奖励重新分堆,再归一化" >}}

{{< figure src="06-bean-tilt.png" alt="小黑把原模型一袋豆子按奖励重新分堆" caption="小黑把 π_ref 这袋豆子按奖励重新分堆:奖励越高堆越大,β 越小分得越极端" >}}

两边取 log、把 r 移到一边,得到「奖励的反解式」:

{{< figure src="fig-kl-invert.png" alt="r = beta log(pi star / pi_ref) + beta log Z" caption="奖励 = β·log(π*/π_ref) + β·logZ。前半截是隐含奖励,后半截 βlogZ 是个麻烦" >}}

前半截正是我们一直用的隐含奖励,但多出个 `β·logZ`——而 Z 要对「所有可能的回答」求和,**根本算不出来**。早期方法就卡死在这。

DPO 的神来之笔:**不去算 Z,只比较同一个问题下的两个回答。** 奖励一相减:

```text
r(y_w) − r(y_l)
 = [ β·log(πθ(y_w)/πref(y_w)) + β·logZ ]
 − [ β·log(πθ(y_l)/πref(y_l)) + β·logZ ]
                                 └──┬──┘
                  同一个问题,βlogZ 一模一样 → 相减抵消!
 = β·log(πθ(y_w)/πref(y_w)) − β·log(πθ(y_l)/πref(y_l))
 = margin
```

{{< figure src="07-z-cancel.png" alt="小黑把两边相同的 Z 划掉" caption="两个回答各顶一块一样的 Z,一相减,小黑一笔划掉——算不出也没关系" >}}

**这就是 DPO 的命根子:** 那个算不出来的归一化项 Z,在「成对相减」里自己抵消了。于是奖励不用单独训练(直接 = β·log(π_θ/π_ref)),把这个差塞进 Bradley-Terry 的 σ、套上 −log,就是 DPO 损失。一个本来要「训奖励模型 + 跑 RL + 估 Z」的庞大流程,坍缩成一条监督式损失——这就是名字里 **Direct** 的来历。

顺带解开两个疑问:**为什么必须成对?** 因为 Z 的抵消靠的就是同一个 x 下两回答相减。**为什么后来 SimPO 敢扔掉 π_ref?** 因为它换了一套隐含奖励的定义,不走这条 Z-抵消的路了。

## 收尾

把整篇缩成一句话带走:

> **log 看「放大还是缩小」,−log 把概率变成「答错就罚」的损失,sigmoid 把分数压成概率;奖励 = β·log(π_θ/π_ref),代进 Bradley-Terry 的 σ(分数差) 套上 −log 就是 DPO 损失;β 是橡皮筋硬度(大了学不动、小了会跑飞);而那个算不出的 Z 在成对相减里自己消失——这就是 DPO 不用训奖励模型的全部秘密。**

下一篇接着用这台天平看 **IPO**:它把 DPO 的「把 margin 推到无穷」换成「推到 `1/(2β)` 就停」,给指针装了个限位器。
