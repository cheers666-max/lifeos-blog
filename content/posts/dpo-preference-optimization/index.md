很多人讲 DPO 一上来就甩损失函数,看着像天书。这篇换个法子,**自顶向下**:先想清楚我们到底在解什么问题(对齐、奖励、reward hacking),再一步步推到那条损失。顺序参考了 Umar Jamil 的《DPO Explained》,每个公式都给你带数字算一遍,配上手绘小黑图。读完你能自己把整条逻辑链从头讲给别人听。

> 「偏好优化家族」系列第一篇。后面接着写 IPO、KTO、SimPO,看每个变体动了哪个零件。

## 0 · 先把符号钉死(开篇速查)

全篇统一用这些符号,看公式卡住随时回来翻。**一个惯例:上标星号 `*` 表示「理想 / 真实的那个」**(如 `r*`、`π*`),是想逼近的目标,不是已知量。

| 符号 | 读法 | 它是什么 |
|---|---|---|
| `x` | — | 问题 / prompt |
| `y_w` | y-winner | 好回答(你圈「赢」的那个) |
| `y_l` | y-loser | 差回答(你圈「输」的那个) |
| `π_θ` | pi-theta | **正在训练的模型**(策略),θ 是它的参数 |
| `π_ref` | pi-ref | **参考模型**:训练前冻结的 SFT 模型,当「橡皮筋」 |
| `π*` | pi-star | 理想的最优策略 |
| `r(x,y)` / `r*` | reward | 给「问题+回答」打的实力分 / 奖励(`*`=理想的那个) |
| `h_w, h_l` | — | 好/差回答的**隐含奖励** `= β·log(π_θ/π_ref)` |
| `margin` | — | `h_w − h_l`,好回答比差回答的奖励高多少(天平指针) |
| `β` | beta | 拴绳硬度 / KL 约束强度(大=保守,小=激进) |
| `σ(z)` | sigmoid | `1/(1+e^(−z))`,把任意分数压成 0~1 的概率 |
| `log` / `e` | 自然对数 / ≈2.718 | log 看「放大还是缩小」 |
| `Z(x)` | 配分函数 | 归一化项(对所有回答求和,算不出来,但会抵消) |
| `KL(·‖·)` | KL 散度 | 量 π_θ 偏离 π_ref 多远 |

## 1 · 先看大局:语言模型 + 对齐是什么

**语言模型在干嘛?** 一句话:看着前面的文字,**预测下一个最可能的词(token)**,吐出来,再接着预测下一个,一个一个往外蹦,直到收尾。它本质是个"超级自动补全"。

**那为什么还要"对齐"?** 因为"会补全下一个词"不等于"会好好回答人"。原始模型可能胡说、可能答非所问、可能学了网上的脏东西。**对齐(alignment)就是把模型调成人更想要的样子**——有用、诚实、无害。

DPO 就是做对齐的一种办法。用一句话+一张图说清它干啥:

**DPO 从头到尾就干一件事:拿人对一对回答的好恶,直接把模型往「好回答」那边掰。** 想象一台天平,好回答搁左盘、差回答搁右盘,指针偏多少叫 **margin**;DPO 一直在把指针往好回答那边推。

{{< figure src="01-balance-scale.png" alt="小黑把天平指针往好回答那边推" caption="小黑站在天平柱上把指针往「好回答」推——这就是 DPO 在做的唯一一件事" >}}

下面就自顶向下,把「为什么要奖励 → 怎么从偏好得到奖励 → 怎么推出这条损失」一节节讲清。

## 2 · 为什么需要「奖励」,以及 reward hacking

要让模型对齐,第一步得能**评判**它的回答好不好——也就是给回答**打个分(奖励 reward)**。

**强化学习(RL)的思路**:把模型看成一个**策略(policy)**,它面对问题生成回答;我们想让它**最大化期望奖励**——多生成高分回答,少生成低分的。听起来很合理。

**但这里有个大坑:reward hacking。** 如果模型只盯着"把分数刷高",它会**钻空子**:输出一堆讨好评分、啰嗦、甚至胡编但"看起来高分"的垃圾。就像学生只为考分答题,会去背套路、写正确的废话,而不是真懂。

所以光有"奖励"不够,我们需要两样东西:

1. **一个靠谱的打分器**——而且这个打分器最好从"人容易给的判断"里学(下一节讲为什么是偏好);
2. **一根拴绳**——别让模型为了刷分跑得离原来的样子太远。这根拴绳,就是后面会出现的 **β / KL 约束**(先记住这个伏笔)。

记住这两点,后面所有数学都是在把它们写成公式。

## 3 · 备三块数学积木:log / −log / sigmoid

往下要算了,先备三块小工具,搞懂它们各自在干嘛,公式就不再是天书。先记一句免得踩坑:margin=0 这点藏着**两个不同的数**——`σ(0)=0.5` 是「概率」,`−log σ(0)=−log(0.5)≈0.69` 才是「损失」,而 `−log(0)` 是**正无穷**(不是 0.5!)。

{{< figure src="fig-log-sigmoid.png" alt="ln(x), -ln(p), sigmoid 三张曲线" caption="三块积木:① log 长什么样 ② −log(p) 当损失 ③ sigmoid 把分数压成概率" >}}

### 积木① — log,其实就是问「放大还是缩小了」

`log` 问的事特别朴素:**这个数是 1 的多少倍?用「放大了还是缩小了」来记。** 训练**前**某句话概率 0.2,**后**涨到 0.4,放大 `0.4÷0.2=2` 倍,`log(2)≈0.69` 正数——更想说了;掉到 0.1 是 `÷2`,`log(0.5)≈−0.69` 负数——更不想说了;没变就是 `log(1)=0`。

> 记住:**>0 放大,=0 没变,<0 缩小,趋近 0 一路掉到负无穷。**

{{< figure src="02-log-ruler.png" alt="小黑在以1为中点的标尺上推概率球" caption="以「1」为中点的标尺:往右推=放大→正,往左=缩小→负" >}}

为啥到处取 log?因为它把**连乘变连加**(`log(a×b)=log a+log b`)。整句概率是每个字连乘,小到电脑都快表示不了;取 log 变连加,不下溢、梯度还好算。这就是为什么写「对数概率 log π」。

### 积木② — −log(p),把它念成「模型有多吃惊」

`p` 是模型给**正确选项**分配的概率,越接近 1 越好。怎么把「p 越大越好」翻成损失?加个 `−log`,念成「这次有多吃惊」:

```text
p = 0.9  (笃定选对)      −ln(0.9) ≈ 0.11   几乎不罚
p = 0.5  (半信半疑)      −ln(0.5) ≈ 0.69   罚一点
p = 0.1  (笃定却选错)    −ln(0.1) ≈ 2.30   狠狠罚
p → 0    (打死认定错的)  −ln(0)   → +∞     罚到天上去
```

**这就是你天天在用的交叉熵**——对了不罚,越自信地答错罚得越狠。

{{< figure src="03-penalty-hammer.png" alt="小黑举的惩罚锤随错得多自信而变大" caption="−log(p) 就是惩罚锤:答对不罚,自信答错锤头越来越大" >}}

### 积木③ — sigmoid,一台压缩机

`σ(z)=1/(1+e^(−z))` 把任意分数压回 0~1 当概率:`σ(0)=0.5`、`σ(2)≈0.88`、`σ(−1)≈0.27`。在 DPO 里,喂给它的分数 z 就是 margin。

## 4 · 打分器从哪来:从偏好到分数 Bradley-Terry

上一节说要个"靠谱打分器"。问题是——**怎么得到分数?**

### 为什么不直接给回答打分?

- **绝对打分,难且不靠谱。** 让人给单个回答打「0~100 分」,因人而异、因时而异,「70 和 72 差在哪」说不清,既不稳定也不可比。
- **相对比较,简单又一致。** 让人选「两个里哪个更好」,认知负担低、结论稳。人天生擅长比大小,不擅长定绝对值。

所以真实流程是:**人只做容易的相对判断 → 攒一堆「A 比 B 好」的对子 → 用 Bradley-Terry 把相对偏好"反推"成绝对分数 `r` → 拿去训练。** Bradley-Terry(给棋手排名的 Elo 是它近亲)就是那台「把好收集的相对偏好,翻译成难直接得到的绝对分数」的机器。

### Bradley-Terry:把「谁赢」变成概率

它的想法:**给每个回答一个隐藏「实力分」 `r*`,好回答赢过差回答的概率只由两者实力分的差决定。**

{{< figure src="fig-bradley-terry.png" alt="Bradley-Terry 公式及其等于 sigmoid 的形式" caption="把两个 e^实力分 做 softmax,等价于把『实力分之差』喂给 sigmoid" >}}

**先学会读 `r*(x, y_w)` 这种写法**(很多人卡这):就是中学的 `f(x)`——括号前是函数名,括号里是塞进去的输入:

```text
   r*  (  x ,  y_w  )
   ↑       ↑     ↑
 函数名   塞进去的两样东西
(打分机)  ① 问题 x  ② 回答 y_w
```

`r*` 是一台**打分机**(函数),把「问题+回答」塞进去吐一个分数。为什么问题也要塞?同一个回答换个问题好坏就变了。那 $e^{r^*(x,y_w)}$ 就是把好回答的分数套上 $e$ 变成正**权重**;上海那行(好回答 3 分、差回答 0 分):

```text
e^{好回答分} = e³ ≈ 20.1     ← 好回答的权重
e^{差回答分} = e⁰ = 1
P(好赢) = 20.1 / (20.1 + 1) = 0.95
```

它就是个**两选一的 softmax**。上下同除 $e^{r_w}$,立刻看出它其实就是 sigmoid:

```text
P = e^{r_w} / (e^{r_w} + e^{r_l}) = 1 / (1 + e^{−(r_w − r_l)}) = σ( r_w − r_l )
```

### 再深一层:分差就是「赔率」,r\* 是一条共享的尺

把 $P = \sigma(r_w - r_l)$ 反过来看赔率:

$$\frac{P(\text{赢})}{P(\text{输})} = e^{\,r_w - r_l}$$

**分差就是「赔率的对数」**;`r*` 给的分,本质是一个回答的「对赌实力」。每拉开 1 分,赢的赔率 ×e(≈2.7 倍):

```text
分差 = 1 → 赔率 ≈ 2.7 : 1 → 赢面 73%
分差 = 2 → 赔率 ≈ 7.4 : 1 → 赢面 88%
分差 = 3 → 赔率 ≈ 20  : 1 → 赢面 95%
```

(正好对上 σ(1)/σ(2)/σ(3)——赔率和概率是同一件事两种说法。)还有更妙的一层:你给的全是零散的「A>B」,**BT 把它们拼成一条统一的「分数尺」**,每个回答在尺上有唯一位置(像从一堆「谁比谁高」反推每个人的身高;Elo 也这么排 rating)。

{{< figure src="09-bt-ruler.png" alt="小黑把一堆 A>B 卡片拼成一条分数尺" caption="BT 的活儿:把散乱的「A>B」拼成一条统一分数尺,r* 就是每个回答在尺上的位置" >}}

那 `r*` 怎么定?用**最大似然估计(MLE)**:让"你标的所有偏好都被排对"的总概率最大,等价于最小化 `Σ −logσ(r_w−r_l)`——**这条就是 RLHF 里训练奖励模型的损失**。RLHF 的常规做法就是先这么训出一个奖励模型,再拿它去跑强化学习。而 **DPO 的野心,是连这个奖励模型都不单独训**——下面就看它怎么做到。

## 5 · RLHF 的真正目标:最大奖励 − β·KL

现在把第 2 节那两句话(要高分、别跑太远)正式写成一个目标函数。这就是 RLHF / DPO 共同的出发点:

```text
max   E[ 奖励 r(y) ]   −   β · KL(π ‖ π_ref)
 π      ↑ 想要高奖励         ↑ 但别离原模型太远(防 reward hacking)
```

左边拉你冲高分,右边那根**拴绳**(KL 项)拽你别跑偏——`KL(π‖π_ref)` 就是「π 偏离原模型 π_ref 多远」的尺,两者一样时为 0。**β 就是这根拴绳的硬度**,也是第 2 节埋的那个伏笔。

{{< figure src="08-beta-knob.png" alt="小黑拧 β 旋钮,橡皮筋时紧时松" caption="β 大→绷紧,贴着原模型;β 小→松,撒丫子跑向高奖励(太小就 reward hacking 跑飞)" >}}

| β | 会发生什么 |
|---|---|
| **大** | 拴绳太硬,几乎不动、学不进偏好。**太大 = 学不动** |
| **小** | 为冲高奖励使劲扭曲分布。**太小 = 跑飞**(复读、退化、reward hacking) |
| **常用** | 多数实现默认 `β=0.1`,区间约 `0.1~0.5` |

> 记反一点:β 在后面 `e^(r/β)` 的**分母**,所以「**越小越猛**」。β 大=贴原模型(保守),β 小=冲高奖励(激进)。reward hacking 正是 β 太小、拴绳太松时发生的。

## 6 · 解这个目标:最优策略 + 算不出来的 Z

上面那个"最大奖励 − βKL"是带约束的优化,它有**闭式解**——**拿 π_ref,给每个回答乘奖励权重 `e^(r/β)`,再除以 Z 归一化:**

{{< figure src="fig-kl-solution.png" alt="pi star = 1/Z pi_ref exp(r/beta)" caption="最优策略 = π_ref 按奖励重新分堆,再归一化" >}}

{{< figure src="06-bean-tilt.png" alt="小黑把原模型一袋豆子按奖励重新分堆" caption="小黑把 π_ref 这袋豆子按奖励重新分堆:奖励越高堆越大,β 越小分得越极端" >}}

拿数字感受 β(原模型 `[0.5,0.3,0.2]`,奖励 `[0,1,2]`):

```text
β = 5   →  π* ≈ [0.43, 0.31, 0.26]   几乎没动,还是原模型
β = 1   →  π* ≈ [0.18, 0.29, 0.53]   开始偏向高奖励
β = 0.2 →  π* ≈ [0.00, 0.01, 0.97]   孤注一掷(太小=跑飞)
```

{{< figure src="fig-beta-effect.png" alt="不同 beta 下的损失曲线和最优分布" caption="左:β 越大损失曲线越陡(小动作就满意→贴 ref);右:β 越小概率越堆向高奖励" >}}

两边取 log、把 r 移到一边,得到「奖励的反解式」:

{{< figure src="fig-kl-invert.png" alt="r = beta log(pi star / pi_ref) + beta log Z" caption="奖励 = β·log(π*/π_ref) + β·logZ。前半截是隐含奖励,后半截 βlogZ 是个麻烦" >}}

前半截 `β·log(π*/π_ref)` 很漂亮,但多出个 `β·logZ`——而 **Z 要对「所有可能的回答」求和**(模型能生成的句子无穷多),**根本算不出来**。早期方法就卡死在这。

## 7 · 临门一脚:Z 抵消 → DPO 损失

DPO 的神来之笔:**不去算 Z,只比较同一个问题下的两个回答。** 奖励一相减:

```text
r(y_w) − r(y_l)
 = [ β·log(πθ(y_w)/πref(y_w)) + β·logZ ]
 − [ β·log(πθ(y_l)/πref(y_l)) + β·logZ ]
                                 └──┬──┘
                  同一个问题,βlogZ 一模一样 → 相减抵消!
 = β·log(πθ(y_w)/πref(y_w)) − β·log(πθ(y_l)/πref(y_l))
 = margin
```

{{< figure src="07-z-cancel.png" alt="小黑把两边相同的 Z 划掉" caption="两个回答各顶一块一样的 Z,一相减,小黑一笔划掉——算不出也没关系" >}}

Z 一抵消,**奖励就不用单独训练了**,它直接等于 `β·log(π_θ/π_ref)`——也就是「模型现在比从前多想说这句多少」,一支态度温度计:

{{< figure src="fig-implicit-reward.png" alt="r = beta log (pi_theta / pi_ref)" caption="隐含奖励 = β·log(训练后概率 ÷ 训练前概率)" >}}

{{< figure src="04-reward-thermometer.png" alt="小黑把温度计插进对话气泡读数" caption="奖励 = 态度温度计:更想说→升,更不想说→降" >}}

把这个差(margin)塞进第 4 节 Bradley-Terry 的 σ,再套上第 3 节 −log,就是 **DPO 损失**:

```text
② Bradley-Terry: 赢家赢的概率 = σ(margin)        ← 积木③
③ margin:        = β(h_w − h_l) = β·log 比之差   ← 隐含奖励
④ 变成损失:      最大化 P(对) = 最小化 −log P(对) ← 积木②
                ─────────────────────────────
   L = −log σ(margin) = −log σ(β(h_w − h_l))
```

{{< figure src="fig-dpo-loss.png" alt="DPO 损失函数" caption="括号里=margin;外面 −logσ 负责「差得不够大就狠罚」" >}}

真拿数字走一遍(β=1)。**已经学得不错那对**(赢家 0.4/0.2、输家 0.1/0.2):

```text
h_w = log(0.4/0.2)=0.69,  h_l = log(0.1/0.2)=−0.69
margin = 1.38 → σ(1.38) ≈ 0.80 → 损失 = −ln(0.80) ≈ 0.22  (不大,基本会了)
```

**还没学会那对**(`h_w=0.10, h_l=0.05`):`margin=0.05 → σ≈0.51 → 损失≈0.67`(明显更大)。同一条公式,会的罚得轻、不会的罚得重。

一个本来要「训奖励模型 + 跑 RL + 估 Z」的庞大流程,坍缩成一条监督式损失——这就是名字里 **Direct** 的来历。顺带:**为什么必须成对?** Z 的抵消靠的就是同一个 x 下两回答相减;**为什么 SimPO 敢扔掉 π_ref?** 它换了一套隐含奖励定义,不走 Z-抵消这条路了。

## 8 · 两个反直觉的坑

**坑 A — 越不会的、推得越狠。** 推力(梯度大小)= `σ(−margin)`:

```text
学得不错那对:  margin = 1.38   推力 = σ(−1.38) ≈ 0.20   轻轻推
还没学会那对:  margin = 0.05   推力 = σ(−0.05) ≈ 0.49   使劲推
要是学反了:    margin = −2     推力 = σ(2)     ≈ 0.88   往死里推
```

{{< figure src="fig-gradient-vs-margin.png" alt="损失与推力随 margin 的变化" caption="推力 σ(−margin):模型越错推得越狠,学会了就放手" >}}

易混点:**标签**是你定死的(谁赢谁输);**margin** 是模型实时算的。margin≈0 **不是**「标签模糊」,而是「模型还没同意你的标注」。DPO **无条件相信标签**——你标错了它照样理直气壮学错,这正是 cDPO、IPO 要打折的地方。

**坑 B — 好回答的概率,其实可能在往下掉。** DPO 靠「拉开差距」满足损失,不保证 `π_θ(y_w)` 真涨。常见的是好坏概率一起掉、只是差的掉更快。这就是有人给 DPO 加 SFT 正则(如 DPOP)的原因。

## 9 · 实战:这些 log 概率到底怎么算

公式里的 `log π(y|x)` 是「整句回答」的对数概率。一句回答有很多 token,**整句概率 = 每个 token 概率连乘**,取 log 后就是**连加**:

```text
log π(y | x) = Σ  log π( 第 t 个 token | 问题 x + 前面已生成的 token )
              t∈答案
```

具体做法:把「问题 + 答案」一起喂进模型,因为答案是数据集里给定的,**每个答案 token 是什么我们都知道**;于是逐个取出"模型在那一步给这个真实 token 的对数概率",求和,就是 `log π_θ(y|x)`。同一句话在 `π_θ` 和冻结的 `π_ref` 上各算一遍,相减,就得到隐含奖励里的那个 `log(π_θ/π_ref)`,winner、loser 各来一遍,一减就是 margin。

举例:问题「上海在哪」,答案「上海是中国的城市」。我们知道答案逐字是什么,就把模型在每个字那一步的 log-prob 加起来——这就是这条回答的 log 概率。整个 DPO 训练,就是在大量这样的成对样本上,最小化 `−logσ(margin)`。

## 收尾

把整篇缩成一句话带走:

> **要对齐模型 → 得给回答打分,但只盯分数会 reward hacking,所以"最大奖励 − β·KL拴绳";人难绝对打分,于是用偏好 + Bradley-Terry 反推分数;解这个带约束目标得到 π*,里面有个算不出的 Z;但成对相减时 Z 自己抵消,奖励就成了 β·log(π_θ/π_ref),套进 σ 和 −log 就是 DPO 损失。Direct 的意思,就是把奖励模型直接吸收进了损失。**

下一篇接着用这台天平看 **IPO**:它把 DPO 的「把 margin 推到无穷」换成「推到 `1/(2β)` 就停」,给指针装了个限位器。
